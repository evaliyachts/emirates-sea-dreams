## Required method for PR 8B

- Record tool/version, production or immutable preview URL, date/time and network/CPU profile.
- Test mobile at 360×800 and 390×844, and desktop at 1440×900; add a narrow 320-pixel overflow check.
- Run at least three Lighthouse passes per representative 200 page and report the median without claiming field data.
- Record total JavaScript, image transfer, DOM size and console errors from the same revision.
- Run an axe-core or equivalent rendered scan on all seven page classes.
- Complete keyboard-only navigation, visible-focus, dialog/menu focus, skip/navigation order and form-label checks.
- Verify `prefers-reduced-motion: reduce` for page transitions, animated sections and the yacht carousel.
- Check mobile overflow and horizontal scrolling in left-to-right layout.
- Verify hero/LCP resource priority and that non-critical images remain lazy-loaded with dimensions.
